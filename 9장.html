왜 보안이 필요한가 — 정보의 3가지 위협

정보를 보내고·받고·맡길 때 우리를 노리는 위험은 크게 세 종류입니다.

도청 (Eavesdropping)

누가 내 메시지를 훔쳐본다

  • 카페 와이파이로 로그인했더니…
  • 같은 망의 누군가가 패킷을 가로챔
  • 해결책 → 암호화

변조 (Tampering)

중간에서 메시지가 바뀐다

  • "100만원 송금"이 "1000만원"으로
  • 파일 다운로드 중 한 비트만 뒤집어도 OS가 오동작
  • 해결책 → 해시

위장 (Impersonation)

상대가 진짜인지 모른다

  • "네이버"라고 쓰여 있지만 가짜 사이트
  • 친구 사칭 메시지로 송금 요구
  • 해결책 → 인증서·서명

오늘 1교시는 그중 변조를 잡아내는 도구 해시(Hash)를 배웁니다. "이 파일이 보낸 그대로인가?"를 한 줄로 확인하는 기술입니다.

정상 시나리오

3
출력된 64자가 SHA256SUMS와 일치하는지 확인

a4acfda10b18d...e7b6 ubuntu-22.04.iso
내 PC 계산:
a4acfda10b18d...e7b6 ubuntu.iso ✓ 일치

1
네트워크 중간자가 ISO에 한 줄 코드 추가
2

3
다행히 sha256sum 계산해보면…

Dockerfile — 이미지 만드는 레시피 (키워드 4개)

텍스트 파일 한 장으로 "내 이미지"를 만든다. 일단 이 4개만 알아도 90%.

Dockerfile
# 1) 기반 이미지 — 무에서 시작하지 않는다
FROM nginx:alpine

# 2) 호스트 파일을 이미지 안으로 복사
COPY ./site/ /usr/share/nginx/html/

# 3) 빌드 시점에 실행할 명령 (선택)
RUN echo "build at $(date)" > /etc/build.txt

# 4) 컨테이너 시작 시 실행할 명령
CMD ["nginx", "-g", "daemon off;"]
FROM
기반 이미지

어떤 이미지에서 출발할지. ubuntu, alpine, node:20, nginx:alpine 등 공식 이미지를 흔히 사용.

COPY
파일 복사

호스트의 파일/폴더를 이미지 안으로 복사 (빌드 컨텍스트). 정적 자산·소스 코드 넣을 때.

RUN
빌드 시 명령

이미지를 만들 때 실행. apt install, npm install 등 환경 구성. 결과가 레이어로 굳어진다.

CMD
실행 시 명령

컨테이너가 시작될 때 돌릴 기본 명령. 1개의 메인 프로세스를 띄우는 게 일반적.

#!/bin/bash
# 라이브 대시보드 생성 루프
cd ~/goodbye

while :; do
  # 1) 실값 추출
  CPU=$(sar 1 1 | grep "평균:" | awk '{print 100-$8}')
  IDLE=$(awk "BEGIN{print 100-$CPU}")
  MUSED=$(free | grep Mem | awk '{print $3}')
  MFREE=$(free | grep Mem | awk '{print $4}')
  DUSE=$(df -h / | tail -1 | awk '{print $5}' | tr -d %)
  DFREE=$(awk "BEGIN{print 100-$DUSE}")

  # 2) t1 + 데이터 + t2 합치기
  cat t1.html > serverstatus.html
  echo "cpuData=[$CPU,$IDLE];" >> serverstatus.html
  echo "memData=[$MUSED,$MFREE];" >> serverstatus.html
  echo "dskData=[$DUSE,$DFREE];" >> serverstatus.html
  cat t2.html >> serverstatus.html

  sleep 1
done
