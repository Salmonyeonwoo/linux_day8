# COMMAND

useradd

리눅스 시스템에 새로운 사용자 계정을 생성하는
저수준(Low-level) 관리 명령어입니다.

-m 홈 디렉터리 자동 생성 (/home/user)
-s 기본 로그인 쉘 지정 (예: /bin/bash)
-u 특정 UID(User ID) 번호 지정
-g 기본 그룹(Primary Group) 지정
-G 보조 그룹(Supplementary Groups) 추가
-c 사용자 설명(Comment) 및 정보 입력
root@server: ~

# 1. 기본 생성 (홈 디렉터리, Bash 쉘)
root@server:~# useradd -m -s /bin/bash -c "Hong GilDong" gil

# 2. 고급 생성 (그룹 먼저 생성 후 UID·그룹 지정)
root@server:~# groupadd dev
root@server:~# useradd -u 1501 -g dev -G docker,sudo -m dev01

# 생성 결과 확인
root@server:~# grep dev01 /etc/passwd
dev01:x:1501:1005::/home/dev01:/bin/sh

Tip: 옵션을 지정하지 않으면 /etc/default/useradd 설정 파일의 기본값을 따릅니다.

# USER MANAGEMENT

usermod & userdel

기존 사용자 계정의 속성을 변경하거나,
더 이상 필요하지 않은 계정을 삭제하는 관리 명령어입니다.

-s 로그인 쉘 변경 (Shell)
-aG 보조 그룹에 추가 (Append + Groups)
-l 사용자 로그인 ID 변경 (Login name)
-L 계정 잠금 (Lock password)
-r 삭제 시 홈 디렉터리도 함께 제거
root@server: ~

# 1. 쉘 변경 (zsh로 변경)
root@server:~# usermod -s /bin/zsh alice

# 2. 보조 그룹 추가 (docker 그룹)
root@server:~# usermod -aG docker alice

# 3. 로그인 ID 변경 (alice -> alice2)
root@server:~# usermod -l alice2 alice

# 4. 사용자 및 홈 디렉터리 완전 삭제
root@server:~# userdel -r tempuser

Warning: userdel 명령만 실행하면 홈 디렉터리가 남습니다. 깔끔한 삭제를 위해 -r 옵션 사용을 권장합니다.

# GROUPS

그룹 관리
개요

리눅스에서 그룹은
효율적인 권한 관리를 위한
핵심 단위입니다.

권한 관리 단위 (GID)

동일한 권한을 공유하는 사용자들의 집합이며, 시스템 내부적으로 GID(Group ID) 숫자로 식별됩니다.

기본 그룹 vs 보조 그룹

파일 생성 시 적용되는 기본 그룹(Primary)과 추가 권한 획득을 위한 보조 그룹(Supplementary)이 있습니다.

파일 권한 연동

ls -l 출력의 그룹 필드와 매칭되어, 그룹 멤버에게 읽기/쓰기 등의 특정 권한을 일괄 부여합니다.

관련 설정 파일

그룹 정보와 멤버 목록은 /etc/group에, 그룹 관리자와 비밀번호는 /etc/gshadow에 저장됩니다.

# GROUP MANAGEMENT

그룹 관리 명령어

groupadd · groupmod · groupdel

사용자들의 권한을 묶어 효율적으로 관리하기 위한
그룹(Group)을 생성, 수정, 삭제하는 도구입니다.

groupadd
새로운 그룹 생성 -g (GID 지정)
groupmod
기존 그룹 속성 변경 -n (이름 변경), -g (GID 변경)
groupdel
그룹 삭제 삭제 전 멤버 확인 권장
root@server: ~

# 1. 새 그룹 생성 (dev)
root@server:~# sudo groupadd dev

# 2. 그룹 이름 변경 (dev -> developers)
root@server:~# sudo groupmod -n developers dev

# 3. GID를 2500으로 변경
root@server:~# sudo groupmod -g 2500 developers

# 4. 이전 그룹 삭제
root@server:~# sudo groupdel oldgroup

Warning: 그룹을 삭제해도 해당 그룹이 소유하던 파일은 삭제되지 않습니다. 파일은 소유 그룹 ID(GID)만 남게 됩니다.

root ALL=(ALL:ALL) ALL   # root는 모든 호스트에서, 임의의 사용자:그룹으로, 모든 명령 실행 가능 (전권)

alice / %group → 권한 부여 대상 (사용자 또는 그룹)
ALL → 적용 호스트 (모든 호스트에서 허용)
(ALL:ALL) → 전환 가능한 사용자:그룹 (모두 = 전권)
(root) → root 권한으로만 실행 허용 (제한)
NOPASSWD: → 비밀번호 입력 생략 옵션

chmod

파일이나 디렉터리의 접근 권한(Mode)을 변경합니다.
숫자 표기법과 기호 표기법을 모두 지원합니다.

NNN 숫자 모드 (예: 755 = rwxr-xr-x)
u+x 기호 모드: 소유자(u)에게 실행(x) 추가(+)
g-w 기호 모드: 그룹(g)에서 쓰기(w) 제거(-)
-R 재귀적(Recursive) 변경: 하위 파일 포함
a=r 모든 사용자(a)에게 읽기(r)만 설정(=)
alice@server: ~/project

# 1. 숫자 모드: 소유자rw, 그룹r, 기타r
alice@server:~$ chmod 644 report.txt

# 2. 기호 모드: 소유자에게 실행 권한 추가
alice@server:~$ chmod u+x run.sh
-rwxr--r-- 1 alice dev 54 run.sh

# 3. 재귀적 변경: 디렉터리 전체 750 설정
alice@server:~$ chmod -R 750 project/

# 4. 권한 제거: 그룹의 쓰기 권한 삭제
alice@server:~$ chmod g-w notes.md

Tip: 기호 모드에서 ugoa (User, Group, Others, All)와 연산자 + - = 를 조합하여 직관적으로 권한을 제어할 수 있습니다.

022 기본값 (파일 644, 디렉터리 755)
027 보안 강화 (파일 640, 디렉터리 750)
alice@linux: ~

# 1. 현재 umask 확인 (기본값 0022)
alice@linux:~$ umask
0022

# 2. 파일 생성 시 권한 확인 (666 - 022 = 644)
alice@linux:~$ touch file1
alice@linux:~$ ls -l file1
-rw-r--r-- 1 alice alice 0 ... file1

# 3. umask 변경 (보안 강화)
alice@linux:~$ umask 027
alice@linux:~$ mkdir secure_dir
alice@linux:~$ ls -ld secure_dir
drwxr-x--- 2 alice alice 4096 ... secure_dir

# 777 - 027 = 750 (그룹 r-x, 기타 ---)

Best Practice: 영구 설정을 위해 ~/.bashrc 또는 /etc/profile에 umask 027을 추가하는 것이 보안상 권장됩니다.

find -exec — 검색 결과로 명령 실행

1

-exec 명령 {} \;

{}는 찾은 파일명. \;로 종료 (이스케이프 필요).

2

{} + 묶음 처리

+는 한 번에 여러 파일 처리 (xargs와 비슷, 더 빠름).

3

-delete 옵션

단순 삭제는 -exec rm보다 -delete가 안전·빠름.

안전: 항상 -name만 먼저 실행해서 결과 확인 후 -delete 추가.

# 1. 30일 지난 로그 압축
$ find /var/log -name '*.log' -mtime +30 \
       -exec gzip {} \;

# 2. 권한 일괄 변경
$ find ~/project -name '*.sh' \
       -exec chmod +x {} \;

# 3. 한 번에 여러 파일 (효율적, +)
$ find . -name '*.bak' -exec rm {} +

# 4. -delete 옵션 (가장 안전)
$ find /tmp -name 'core.*' -mtime +7 -delete

# 5. 확인 후 실행 (-ok, 매 파일마다 y/n)
$ find ~ -name '*.tmp' -ok rm {} \;
# < rm ... ./old.tmp > ? y

# 6. 검색 + grep 조합
$ find ~/project -name '*.py' -exec grep -l 'TODO' {} +

# 7. 크기 큰 파일 Top 10
$ find . -type f -exec ls -lh {} + \
       | sort -k5 -rh | head -10

# 8. xargs 패턴 (대안)
$ find . -name '*.log' | xargs gzip
